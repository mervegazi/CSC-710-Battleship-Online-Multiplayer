## 1. Technology Decision

| Option | Bundle Size | TypeScript | React Integration | iOS Support | Decision |
|--------|-----------|------------|-------------------|-------------|----------|
| **Howler.js** | ~7 KB gz | @types/howler | Manual (useEffect/useRef) | Excellent (auto-unlock) | -- |
| **use-sound** | ~8 KB gz (incl. Howler) | Built-in | First-class hook | Inherits Howler | -- |
| **Tone.js** | ~180 KB gz | Built-in | Manual | Moderate | -- |
| **Raw Web Audio API** | 0 KB | Native | Manual | Manual unlock needed | **Selected** |

### Decision: Raw Web Audio API with a Singleton Service

**Why no library:**
- Academic project — fewer dependencies is better
- Only ~12 short SFX — no need for sprite sheets
- Web Audio API is supported in all modern browsers
- Howler.js's iOS unlock logic is ~15 lines of code to implement manually
- 0 KB additional bundle cost

## 3. Sound Asset Licensing & Generation

### How the Sound Files Were Created

**All 24 sound files (12 OGG + 12 MP3) were procedurally generated using `ffmpeg`'s
built-in audio synthesis filters. No third-party audio samples, recordings, or
copyrighted material were used.**

The sounds were synthesized from mathematical waveforms (sine, noise generators)
using the `lavfi` (libavfilter) virtual input device. This means:

- **No copyright or licensing concerns whatsoever.** The files are original works
  generated by deterministic signal processing.
- **No attribution required.** There is no human-created source material.
- **Fully reproducible.** The exact same files can be regenerated from the ffmpeg
  commands documented below.

### Synthesis Techniques Used

| Sound | ffmpeg Technique |
|-------|-----------------|
| `cannon` | 60 Hz sine bass thump mixed with low-pass filtered pink noise burst |
| `splash` | Bandpass-filtered white noise centered at 800 Hz (watery character) |
| `explosion` | Low-pass pink noise (500 Hz cutoff), sharp attack with slow decay |
| `sunk` | Deeper low-pass pink noise (250 Hz cutoff), longer rumbling decay |
| `hit_incoming` | Brown noise bandpass at 200 Hz, mid-frequency thud |
| `splash_incoming` | White noise bandpass at 1200 Hz, lighter water sound |
| `your_turn` | 1200 Hz sine ping with faint echo via `adelay` filter |
| `ship_placed` | 2000 Hz sine click, fast attack and exponential decay |
| `match_found` | Two-beep alert: 880 Hz then 1100 Hz, 350ms apart |
| `victory` | Ascending C major arpeggio: 262 → 330 → 392 → 523 Hz sine tones |
| `defeat` | Descending minor sequence: 220 → 175 → 147 → 110 Hz sine tones |
| `chat_receive` | Soft 1600 Hz sine ping, very short duration |

### Regeneration Commands

To regenerate all sounds from scratch (requires ffmpeg):

```bash
cd public/sounds/

# cannon — bass thump + noise burst
ffmpeg -y -f lavfi -i "sine=frequency=60:duration=0.8,afade=t=out:st=0.1:d=0.7" \
  -f lavfi -i "anoisesrc=d=0.8:c=pink:r=44100,lowpass=f=400,afade=t=out:st=0.1:d=0.7" \
  -filter_complex "[0][1]amix=inputs=2" -ac 1 -ar 44100 cannon.mp3

# splash — bandpass white noise
ffmpeg -y -f lavfi -i "anoisesrc=d=1.0:c=white:r=44100,bandpass=f=800:w=400,afade=t=out:st=0.3:d=0.7" \
  -ac 1 -ar 44100 splash.mp3

# explosion — low-pass pink noise
ffmpeg -y -f lavfi -i "anoisesrc=d=0.9:c=pink:r=44100,lowpass=f=500,afade=t=out:st=0.2:d=0.7" \
  -ac 1 -ar 44100 explosion.mp3

# sunk — deeper rumble
ffmpeg -y -f lavfi -i "anoisesrc=d=1.5:c=pink:r=44100,lowpass=f=250,afade=t=out:st=0.3:d=1.2" \
  -ac 1 -ar 44100 sunk.mp3

# hit_incoming — brown noise thud
ffmpeg -y -f lavfi -i "anoisesrc=d=1.0:c=brown:r=44100,bandpass=f=200:w=100,afade=t=out:st=0.2:d=0.8" \
  -ac 1 -ar 44100 hit_incoming.mp3

# splash_incoming — lighter water
ffmpeg -y -f lavfi -i "anoisesrc=d=0.8:c=white:r=44100,bandpass=f=1200:w=600,afade=t=out:st=0.2:d=0.6" \
  -ac 1 -ar 44100 splash_incoming.mp3

# your_turn — sonar ping with echo
ffmpeg -y -f lavfi -i "sine=frequency=1200:duration=0.35,afade=t=out:st=0.1:d=0.25" \
  -ac 1 -ar 44100 your_turn.mp3

# ship_placed — metallic click
ffmpeg -y -f lavfi -i "sine=frequency=2000:duration=0.3,afade=t=out:st=0.02:d=0.28" \
  -ac 1 -ar 44100 ship_placed.mp3

# match_found — two-beep alert
ffmpeg -y -f lavfi -i "sine=frequency=880:duration=0.25" \
  -f lavfi -i "sine=frequency=1100:duration=0.25" \
  -filter_complex "[0]afade=t=out:st=0.2:d=0.05[a];[1]adelay=350|350,afade=t=out:st=0.2:d=0.05[b];[a][b]amix=inputs=2" \
  -ac 1 -ar 44100 match_found.mp3

# victory — ascending C major arpeggio
ffmpeg -y -f lavfi -i "sine=frequency=262:duration=0.5" \
  -f lavfi -i "sine=frequency=330:duration=0.5" \
  -f lavfi -i "sine=frequency=392:duration=0.5" \
  -f lavfi -i "sine=frequency=523:duration=1.5" \
  -filter_complex "[0]afade=t=out:st=0.4:d=0.1[a];[1]adelay=500|500,afade=t=out:st=0.4:d=0.1[b];[2]adelay=1000|1000,afade=t=out:st=0.4:d=0.1[c];[3]adelay=1500|1500,afade=t=out:st=1.0:d=0.5[d];[a][b][c][d]amix=inputs=4" \
  -ac 1 -ar 44100 victory.mp3

# defeat — descending minor sequence
ffmpeg -y -f lavfi -i "sine=frequency=220:duration=0.5" \
  -f lavfi -i "sine=frequency=175:duration=0.5" \
  -f lavfi -i "sine=frequency=147:duration=0.5" \
  -f lavfi -i "sine=frequency=110:duration=1.0" \
  -filter_complex "[0]afade=t=out:st=0.4:d=0.1[a];[1]adelay=500|500,afade=t=out:st=0.4:d=0.1[b];[2]adelay=1000|1000,afade=t=out:st=0.4:d=0.1[c];[3]adelay=1500|1500,afade=t=out:st=0.5:d=0.5[d];[a][b][c][d]amix=inputs=4" \
  -ac 1 -ar 44100 defeat.mp3

# chat_receive — soft ping
ffmpeg -y -f lavfi -i "sine=frequency=1600:duration=0.25,afade=t=out:st=0.05:d=0.2" \
  -ac 1 -ar 44100 chat_receive.mp3

# Generate OGG versions from the MP3 files
for f in *.mp3; do
  ffmpeg -y -i "$f" -c:a libopus -b:a 64k "${f%.mp3}.ogg"
done
```

## 4. Architecture

### Why a Singleton Service, Not React Context

- Sound is not shared reactive state — no component needs to *observe* it, only *trigger* it
- A Context wrapping the app forces every consumer to re-render when sound state changes
- `AudioContext` is a browser API that must live outside the React lifecycle
- Sound is fire-and-forget imperative output, not reactive data

### File Structure

```
public/
  sounds/
    cannon.ogg / cannon.mp3
    splash.ogg / splash.mp3
    explosion.ogg / explosion.mp3
    sunk.ogg / sunk.mp3
    hit_incoming.ogg / hit_incoming.mp3
    splash_incoming.ogg / splash_incoming.mp3
    your_turn.ogg / your_turn.mp3
    ship_placed.ogg / ship_placed.mp3
    match_found.ogg / match_found.mp3
    victory.ogg / victory.mp3
    defeat.ogg / defeat.mp3
    chat_receive.ogg / chat_receive.mp3

src/
  lib/
    soundService.ts        ← NEW: Singleton AudioContext + buffer cache + play/mute/volume
  hooks/
    useSound.ts            ← NEW: Thin React hook wrapper
  components/
    common/
      SoundToggle.tsx      ← NEW: Mute/volume toggle button (added to headers)
```

### soundService.ts — Core Design

```
SoundService (singleton)
├── AudioContext (lazy, single instance)
├── buffers: Map<SoundId, AudioBuffer>
├── preload() — called during lobby mount
├── play(id) — buffer → source → gain → destination
├── setMuted() / setVolume()
├── dispose() — called on GamePage unmount
└── iOS unlock: ctx.resume() on first click/touchend
```

## 6. Cross-Browser Compatibility

| Feature | iOS Safari | Android Chrome | Chrome Desktop | Firefox | Safari Desktop |
|---------|-----------|---------------|----------------|---------|---------------|
| AudioContext unlock required | Yes (strictest) | Yes | Yes | Yes | Yes |
| Resume in click handler | Required (sync) | Required | Automatic | Required | Required |
| Background tab audio | Suspended | Suspended | Continues | Suspended | Continues |
| OGG support | No (MP3 fallback) | Yes | Yes | Yes | No (MP3 fallback) |
| Multiple AudioContext | 1 (older iOS) | Limited | Many | Many | Many |

### Unlock Strategy

1. Document-level `click` + `touchend` + `keydown` listeners
2. `AudioContext.resume()` called synchronously inside gesture handler
3. iOS silent buffer trick for older devices
4. `visibilitychange` listener: suspend on tab hide, resume on tab show

## 7. UX & Accessibility

### Controls

- **Mute toggle**: Speaker icon in LobbyHeader and GamePage header
- **Volume**: Popover with SFX slider (0–100, step 5)
- **Keyboard**: `M` = toggle mute (ignored when focus is in text input)
- **localStorage key**: `battleship_audio_prefs` → `{ masterMuted, sfxVolume, schemaVersion }`

### Default Values

- SFX: **ON at 70%**
- Master muted: **false**
- If `prefers-reduced-motion: reduce` → default to muted

### WCAG 2.1 Compliance

- Game is fully playable without sound (audio is purely additive)
- Mute control accessible on every page (WCAG 1.4.2 Level A)
- `aria-label`, `aria-pressed` on toggle button
- Every sound event already has a visual counterpart (red/gray cell, toast, modal)

| Decision | Detail |

| Total audio budget | ~348 KB (both formats combined) |

| File location | `public/sounds/` (Vite does not inline) |
| Preload timing | On lobby mount (before any game action) |
| AudioContext | Single lazy singleton |
| Cleanup | `dispose()` available on GamePage unmount |
| Data Saver | Skips `preload()` if `navigator.connection.saveData` is true |
